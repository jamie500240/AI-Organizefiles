# 🛡️ Script_SafeRenamer (依規則重新命名大量檔名)
## 用可設定的規則批次改檔名，改壞可以復原

Script_SafeRenamer 是一款以「**資料安全與原子性**」為最高指導原則的批次改名工具。採用「先複製、驗證、再保留原檔」的非破壞性設計，並支援強大的 JSON 管線 (Pipeline) 擴充功能與「時光機」無損復原機制。

## 📦 壹、基本環境建置

本程式主要依賴 Python 內建標準庫，力求最低的環境依賴負擔。

1. **系統需求：** Python 3.8 或以上版本（依賴 `Pathlib` 與 `unlink(missing_ok=True)` 語法特性）。
2. **安裝可選套件（強烈建議）：**
若需使用「讀取照片 EXIF 拍攝時間」功能，請安裝 `Pillow` 影像處理套件。
*若未安裝，程式仍可正常運行，但會自動略過 EXIF 相關的改名處理。*
```bash
pip install pillow

```

## 🧩 貳、可用的加工規則總覽 (7 大積木庫)

無論你是使用 Python 預設值還是 JSON，本工具的核心都是由以下 7 種「規則積木」自由組合而成。

| 規則名稱 (Rule) | 功能說明 | 可設定參數 (Params) | 效果範例 |
| --- | --- | --- | --- |
| **`UnderscoreStripRule`** | 依據指定符號切斷檔名，並移除左側或右側的段落。 | `left_strip` (整數), `right_strip` (整數), `sep` (字串，預設 `_`) | `v1_Project_Final.pdf` ➔ `Project_Final.pdf` (切左1) |
| **`RegexRule`** | 萬用殺手鐧。使用正規表達式抓取文字並替換/重組。 | `pattern` (正則字串), `replacement` (替換字串) | `[1080p] Video.mp4` ➔ `Video.mp4` |
| **`DictReplaceRule`** | 字典精準對應替換，可同時設定多組文字轉換。 | `mapping` (字典結構 `{"舊字": "新字"}`) | `Draft` ➔ `v1`, `(空白)` ➔ `_` |
| **`CasingRule`** | 強制轉換英文字母的大小寫。 | `mode` (`upper`, `lower`, `title`) | `FILE_NAME.jpg` ➔ `file_name.jpg` (lower) |
| **`ZeroPadRule`** | 偵測檔名中的數字並自動補滿零，解決作業系統排序錯亂。 | `pad_length` (整數，預設 `3`) | `Report_1.pdf` ➔ `Report_001.pdf` |
| **`OSModifiedTimeRule`** | 讀取檔案最後修改時間，並加在檔名最前面。 | `time_format` (時間格式，預設 `%Y%m%d_`), `prefix` (前綴字串) | `Data.csv` ➔ `20240315_Data.csv` |
| **`ExifDateRule`** | 讀取照片真實拍攝時間。非圖片或無 EXIF 則自動略過。 | `time_format` (時間格式，預設 `%Y%m%d_%H%M%S_`) | `IMG001.jpg` ➔ `20240315_143000_IMG001.jpg` |

## ⚙️ 參、設定與選配方式 (兩種模式任選)

你可以依據使用情境的複雜度，選擇「免設定內建預設」或「外部 JSON 設定」。

### 模式 A：懶人內建預設 (修改 Python 原碼)

適合固定且單純的需求（例如：永遠只切掉底線右邊的字）。完全不需要處理 JSON。
用編輯器打開 `Script_SafeRenamer.py`，找到大約第 110 行的 `DEFAULT_PIPELINE` 變數：

```python
# 將你想要的規則積木放進陣列中，由上往下執行
DEFAULT_PIPELINE: List[RenameRule] = [
    # 範例 1：切除最右邊的 1 個底線與其後面的文字
    UnderscoreStripRule(left_strip=0, right_strip=1, sep="_")
]

```

**【啟用方式】：** 執行程式時，當跳出「請選擇外部 JSON 設定檔」的視窗時，**直接按下【取消】或【X】**，程式就會自動套用內建預設值。

### 模式 B：外部 JSON 引擎 (高擴充性)

適合有多種情境（如 `整理照片.json`、`整理報表.json`）的需求。將規則寫入 JSON 檔案中，執行程式時於視窗選取該檔案即可載入。

## 📖 肆、JSON 設置新手防呆指南

JSON 是一種對格式「極度嚴格」的資料格式，多一個逗號或少一個引號都會導致程式崩潰讀不到。請務必詳讀以下三大重點：

### 1. JSON 必考基本語法

* **雙引號絕對原則：** 所有的文字（包含 `rule` 的名稱、參數的鍵值）都必須使用**雙引號 `""**`，絕對不能用單引號 `''`。
* **致命的逗號陷阱 (Trailing Comma)：** 陣列 `[` `]` 或物件 `{` `}` 裡面的項目之間要用逗號分隔，但**最後一個項目後面絕對不能加逗號**！這是新手最常犯的錯。
* **反斜線跳脫 (Escape)：** 如果你使用 `RegexRule`，正規表達式裡的 `\` 在 JSON 裡必須打兩次變成 `\\`（例如 `\d` 要寫成 `\\d`），否則會解析失敗。

### 2. 由上至下的加工慘劇 (狀態耦合)

**「下一個規則的輸入，是上一個規則的輸出。」**
管線是**由上往下**依序處理的，排錯順序會引發災難。

> **💣 慘劇還原示範：**
> 假設你的原始檔名是 `vacation.jpg`，你同時啟用了「加 EXIF 日期」與「切除最右邊底線」：
> * **步驟 1 (`ExifDateRule`)：** 檔名變成 `20240315_vacation.jpg`
> * **步驟 2 (`UnderscoreStripRule`, right_strip=1)：** 程式尋找最右邊的底線，並切掉後面的文字。結果檔名變成 `20240315.jpg`！
> * **慘劇結果：** 你原始的 `vacation` 關鍵字直接被吃掉了。
> 
> 

**✅ 破解心法：三階段選配法**
請嚴格遵守以下排列順序，絕不打架：

1. **Phase 1 (清理與破壞)：** 先切除不要的雜訊 (`UnderscoreStripRule`, `RegexRule` 移括號)。
2. **Phase 2 (轉換與正規化)：** 幫留下來的文字做美容 (`DictReplaceRule`, `CasingRule`, `ZeroPadRule`)。
3. **Phase 3 (賦予中繼資料)：** **最後一步**才加上系統時間或照片 EXIF (`OSModifiedTimeRule`, `ExifDateRule`)。因為加了時間前綴會改變檔名結構，絕對不能提早加。

### 3. 如何新增與停用範例規則

我們隨附的 `rename_config_tutorial.json` (新手教學版) 內含了所有的可用規則與備用模板。

* **如何新增 (啟用) 規則？**
不用從頭寫！直接複製一份教學版 JSON。如果你想加入一個新規則，請將整個大括號 `{ ... }` 區塊複製並插入 `pipeline` 陣列中，並注意**區塊之間要有逗號**。
* **如何停用規則 (X 前綴封印法)？**
千萬別直接把那段程式碼刪掉（以後要用會找不到）。你只需要在 `rule` 的名稱最前面加上大寫的 **`X`**：
* 啟用狀態：`"rule": "RegexRule"`
* 封印狀態：`"rule": "XRegexRule"`
程式看到 `X` 開頭，會知道這是你刻意停用的規則，便會顯示 `[停用]` 並安全略過，留作未來備用。

## 🚀 伍、完整使用方式 (執行流程)

啟動程式：

```bash
python Script_SafeRenamer.py

```

### [模式 1] 執行批次改名 (Rename Mode)

1. **載入設定：** 終端機輸入 `1`。跳出視窗選擇 JSON 檔（按取消則使用內建預設）。
2. **選擇路徑：** 依序選擇「來源資料夾」(要處理的原始檔) 與「目的資料夾」(完成後的存放區)。**兩者不可相同。**
3. **Dry Run (安全預演)：** 終端機會印出所有即將發生的檔名變化，並估算複製所需時間。*此階段不會更動硬碟資料。*
4. **確認執行：** 輸入 `Y`。程式會先複製檔案，執行大小驗證，成功後寫入 `rename_log.txt`，並在目的資料夾產生一份時光機地圖檔 `revert_map_YYYYMMDD_HHMMSS.json`。

### [模式 2] 啟動時光機 (Revert Mode)

1. **啟動模式：** 終端機輸入 `2`。
2. **選擇地圖檔：** 於視窗中選擇前次執行產生的 `revert_map_YYYYMMDD_HHMMSS.json`。
3. **確認還原：** 程式會列出即將還原的檔案對應表。輸入 `Y` 後，程式將直接在「目的資料夾」內，使用 OS 底層高效率的 `rename` 動作將檔案完美更名回原始名稱。萬一原檔名被佔用，程式會自動加上 `_revert1` 尾綴避免覆蓋。

## ⚠️ 陸、使用限制與安全邊界

1. **1:1 硬碟空間需求：**
基於最高安全考量，本工具採「複製」而非搬移。執行批次改名時，目的硬碟**必須具備至少等於來源檔案總大小的剩餘空間**。空間不足將觸發強制回滾 (All-or-Nothing)，自動清空寫入一半的殘檔。
2. **檔名合法性強制過濾 (Sanitization)：**
所有經過管線處理後產生的新檔名，若包含 Windows 檔案系統的非法字元 `< > : " / \ | ? *`，將在最後一關被系統強制替換為底線 `_`，防止複製階段系統報錯崩潰。
3. **白名單安全機制：**
外部 JSON 引擎採用嚴格的白名單 (`_REGISTRY`)。任何未註冊在上述 7 大規則內的指令，都會被系統安全略過並顯示警告，杜絕透過 JSON 注入任意惡意程式碼的資安風險。
4. **子資料夾不遞迴 (Non-recursive)：**
為避免路徑結構混亂，程式採用「平坦化處理」，僅讀取來源資料夾「第一層」的檔案，不會主動鑽入子資料夾讀取。
5. **純文字介面與 GUI 混合依賴：**
程式流程於終端機 (CLI) 運行，但路徑與檔案選擇依賴 `tkinter` 呼叫原生作業系統視窗。因此無法於無桌面環境的純文字伺服器 (Headless Server) 上執行。
