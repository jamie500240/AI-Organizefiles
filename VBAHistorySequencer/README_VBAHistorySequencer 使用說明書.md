# 🧬 VBA History Sequencer (幫雜亂的 VBA 定出時間序)
## 從一堆 VBA 巨集舊檔中，反推版本演化順序
**版本：** 1.1.0 [Experimental]

**用途：** 針對大量包含 VBA 巨集的 Excel 歷史檔案，進行自動化原始碼提取、血緣追蹤、去重隔離與變更分析。本工具專為處理數百 GB、成千上萬個歷史版本的極端情境所設計，具備 OOM 防禦與系統級防呆機制。

---

## 🛠️ 1. 環境建置 (Environment Setup)

本工具為純 Python 腳本，無需安裝繁雜的資料庫，但**極度依賴特定的解析引擎**。

### 步驟 1：安裝 Python

請確保你的電腦已安裝 Python 3.8 或以上版本。

### 步驟 2：安裝必備與加速套件 (重要！)

請打開終端機 (CMD / PowerShell) 並執行以下指令：

```bash
pip install oletools rapidfuzz

```

#### ⚠️ 為什麼你需要裝這兩個？（非直覺陷阱）

* **`oletools` (必裝)**：核心的二進位 VBA 解析器。沒裝這個，程式一秒都不會跑。
* **`rapidfuzz` (強烈建議)**：程式內部設計了「優雅降級」機制，如果你沒裝這個套件，程式依然會跑，但會默默切換成 Python 內建的 `difflib` 來比對巨集相似度。**未安裝時執行效能會明顯下降，當檔案數量越多、程式碼越長時，落差會非常顯著**。請務必安裝以開啟「極速模式」。

## 🚀 2. 使用方式 (Usage Flow)

直接雙擊執行腳本，或在終端機輸入 `python script_name.py`。
程式啟動後，**不會有複雜的視窗介面，而是依序彈出「三個對話框」**。請依照以下邏輯進行選擇：

### 彈窗一：【選擇全庫來源】

* **你要選什麼：** 包含你所有新舊 Excel 檔案（`.xls`, `.xlsx`, `.xlsm`, `.xlsb`）的最上層資料夾。
* **⚠️ 重要限制 (不會掃描子資料夾)：** 程式**只會掃描你選擇的「第一層」目錄**。如果你有巢狀的資料夾結構（例如按年份或專案分子資料夾），裡面的檔案不會被自動讀取。請務必先將所有要比對的 Excel 檔案集中放在同一個平坦的資料夾中。另外，系統會自動忽略非 Excel 檔（如圖片或 PDF）。

### 彈窗二：【指定進化終點 (Anchor File)】 🤯 *(最不直覺的部分)*

* **你要選什麼：** 在剛剛的資料夾中，選出一個「最終定稿」或「最標準/最新版」的單一 Excel 檔案。
* **為什麼要這樣做？** 本工具的運作邏輯是「反推歷史」。它需要一個「終點（Anchor）」作為基準（100% 基因），然後去比對資料夾裡所有的檔案「跟這個終點有多像」，藉此排出世代 (Generation) 順序。如果你選錯了，相似度與演化順序的計算就會全部失準。

### 彈窗三：【選擇輸出位置】

* **你要選什麼：** 決定最後抽出來的巨集檔案與報表要存在哪裡。
* **注意：** 對話框**預設會打開在「來源資料夾的上一層」**，你可以直接點擊確認來使用這個路徑，或是手動指定其他位置。建議選擇一個全新的空資料夾，避免輸出檔案與你原本的資料混在一起。（若直接取消關閉對話框，程式將中止執行）。

## 📂 3. 產出結構與報表解讀 (Understanding the Output)

程式跑完後，你會在輸出位置看到一個名為 `抽取巨集_YYYYMMDD_HHMMSS` 的資料夾。裡面的結構與命名邏輯如下：

### ① 世代資料夾 (`Gen001_...`)

每個被成功解析的 Excel 檔都會獲得一個專屬資料夾，命名格式為：
`Gen[世代順序]_[與最終版的相似度]%_[安全檔名]`
*(例如：`Gen003_85.2%_2023報表_v2`)*

資料夾裡面會裝著從該 Excel 抽出來的 VBA 模組（`.txt` 檔）。

### ② 檔案前綴詞 `[DUP]`

點進世代資料夾，你會看到兩種檔案：

* `Module1_a1b2c3d4.txt`：**MUTATION (突變/首創)**。這代表這段程式碼在這個演化史中是「第一次出現」，或是發生了新的修改。
* `[DUP]_Module1_f9e8d7c6.txt`：**INHERITED (繼承)**。只要檔名開頭有 `[DUP]`，代表這段程式碼跟舊版本的某個模組 **100% 一模一樣**。你可以直接略過不看，節省 Code Review 的時間。

### ③ 特殊隔離區

* **`_CLONES_隔離區`**：如果某個 Excel 檔裡面的「所有巨集」都跟歷史上的某個檔案完全一樣（連一個字都沒改），整個檔案會被判定為純複製體 (Clone)，丟進這裡，不會幫它產生世代資料夾。
* **`_FAILED_解析失敗`**：檔案損毀、被密碼保護，或是**因為老舊格式 (`.xls`) 且容量超大（> 20MB）觸發防禦機制**的檔案，會被原封不動丟進這裡，避免程式崩潰。（註：`.xlsx` 等新版壓縮格式無此 20MB 限制）。

### ④ 結算報表 (`_LOG_Evolution_Tree.csv`)

這是一份流式寫入 (Streaming Write) 的血緣追蹤表。強烈建議用 Excel 打開它並加上篩選功能。你可以透過這份表一眼看出哪個模組在哪個版本發生了修改。

## 🛑 4. 中斷與救援機制 (ADR-001 合約)

處理幾百 GB 的 Excel 可能需要幾十分鐘甚至數小時。
如果你中途發現設錯了，或者想強制關閉：

* **按下 `Ctrl + C` (強烈建議的中斷方式)**：
程式**不會**把你前面跑的成果刪除。它會立即停止後續的解析，並保留當下所有的實體檔案與 CSV 報表。報表中未處理的檔案不會出現，但已處理的資料具有 100% 稽核效力。
* **檔案鎖定異常 (PermissionError)**：
如果處理到一半，某個檔案剛好被別的使用者或系統鎖定，程式不會崩潰，而是將該檔案標記為 `PARSE_FAILED` 寫入報表，並自動繼續處理下一個檔案。
