# 初學者用 AI 寫的 PYTHON 工具箱

我是個初學程式語言的人，完全不懂 Python，這些工具是用 AI 輔助撰寫而成。功能都是圍繞「檔案整理與安全處理」這個主題發展出來的，每支工具獨立運作，互不依賴，各自解決一個具體的痛點。

每個資料夾內都有各自的 README，說明詳細使用方式與環境建置步驟。這份文件只做總覽，方便快速判斷「我現在的狀況該用哪一支」。

## SafeRenamer — 安全批次改名

**解決什麼問題**：檔名亂七八糟（一堆「新增文件」「IMG_001」「草稿_final_真的最終版」），想批次規則化命名，但又怕改壞，改壞想復原。

**使用情境**：
- 整理下載資料夾、掃描歸檔文件，需要統一格式（去掉底線雜訊、日期格式標準化、補零方便排序）
- 需要不同專案套用不同改名邏輯，用 JSON 設定檔切換規則組合，不用改程式碼
- 不確定規則會不會改壞名字，想要「先試跑再確認」

**核心設計**：
- 規則採積木式（UnderscoreStrip、Regex、字典替換、大小寫轉換、補零、EXIF 拍攝時間、檔案修改時間），可依序組合、用 JSON 開關個別規則
- 雙模式：改名模式 + 時光機復原模式，改名同時會產生 `revert_map.json`，復原時直接讀這份地圖一鍵改回原名
- 來源檔案全程只複製不搬動，原始檔案不會被動到；改名發生在複製出去的副本上
- SHA256 雜湊驗證複製完整性，中斷（Ctrl+C）也保證已完成部分的復原地圖會落地，不會產生「改了但復原不了」的檔案

## GodCleanup — 檔案清理去重

**解決什麼問題**：一堆重複備份、雜亂堆積的資料夾，想去重、想整理，但資料量可能上百 GB，怕空間不夠、怕中途出錯把資料搞壞。

**使用情境**：
- 清理多年累積的下載資料夾、隨手複製的重複備份
- 需要先知道「這次整理大概要多久、需要多少額外空間」再決定要不要動手
- 想要去重後把瑣碎的巢狀資料夾自動拍平

**核心設計**：
- 執行前先做 Dry Run：掃描檔案數、總容量、預估所需空間與時間，空間不足會直接擋下
- SHA256 雜湊比對去重，唯一檔案跟重複檔案分開歸檔
- 複製＋驗證雜湊一致才視為成功，多次重試機制應對檔案暫時被鎖定的情況
- 選配：去重後移除比對用的雜湊印記、將檔案數過少的資料夾自動攤平合併

## ArchiveAudit — 歸檔完整性稽核

**解決什麼問題**：東西搬完、歸檔完之後，想知道「有沒有漏搬」或「有沒有撞名覆蓋掉」，需要事後驗收。

**使用情境**：
- 用其他工具（或手動）搬完一批檔案後，想確認原始資料夾與歸檔後資料夾內容是否完全對得上
- 懷疑某次搬移過程中可能漏了檔案，需要一份可稽核的比對報表

**核心設計**：
- 對兩個資料夾各自計算所有檔案的雜湊值，逐一比對，分類為「內容一致」「內容一致但檔名不同（優化）」「歸檔遺漏」「歸檔新增」
- 純比對，不做任何搬移或刪除動作，安全性最高
- 輸出 CSV 報表，遺漏項目會在終端機以警示色標出

## EnglishFileMigrator — 英文檔案分流

**解決什麼問題**：中英文檔案混雜在同一批資料裡，想把英文資料自動挑出來獨立存放。

**使用情境**：
- 需要把英文文件（合約、論文、原文資料）跟中文檔案分開管理
- 檔案格式多樣（PDF、Word、Excel、PPT、圖片皆可），不想逐一手動判斷語系

**核心設計**：
- 依實際檔案內容（非僅檔名）判定語系，支援 PDF / DOCX / TXT / CSV / XLSX / PPTX，圖片可選配 OCR 辨識
- 圖片預設不搬移（保護相簿與資料夾結構完整），需明確開啟才會處理
- 複製＋雜湊驗證＋成功才刪除來源，過程崩潰或中斷會把未完成報表另存為 `.crashed` 保留紀錄
- 已知限制：內容解析失敗（如密碼保護檔案、掃描影像 PDF）會保留原位並在報表註記原因，不強行猜測

## OmniArchiveMigrator — 全能命名歸檔

**解決什麼問題**：檔名本身沒有意義（「新增文件1」「文件(2)」之類），想依內容自動生成有意義的檔名並分流保存。

**使用情境**：
- 整理長期堆積、命名習慣混亂的舊檔案
- 想要「原本命名良好的維持原樣，命名差的才自動改名」，而不是全部一視同仁重新命名

**核心設計**：
- 先判斷檔名是否「醜」（過短、含「新增」「document」等雜訊字樣），醜的才觸發改名
- 從檔案內容（前幾段文字、試算表首格、簡報首頁）擷取中文關鍵字作為新檔名素材
- MD5 去重，內容重複的檔案會標記略過而非重複保存
- 內容辨識失敗（空白頁、純圖片、密碼保護）會誠實標記「內容辨識失敗」，不會編造檔名

## MediaCollapseMigrator — 圖片分發整併

**解決什麼問題**：大量照片散落在深層巢狀資料夾，想依日期或人物分類集中，並清掉一堆只有一兩張照片的瑣碎子資料夾。

**使用情境**：
- 手機／相機備份的照片經年累月後，資料夾結構又深又亂
- 想要有 AI 產出圖片（截圖、生成圖）跟一般照片自動分開的初步判斷（依熵值粗略辨識）

**核心設計**：
- 依 EXIF 拍攝時間或檔案時間分類，若路徑本身已有人名資料夾則優先依人名歸類
- 「貪婪坍塌」演算法：遞迴清理沒有內容或內容過少的資料夾，attempts 到 5 輪，並具備原子性回滾（過程失敗會把已搬移的檔案退回原位）
- 移動一律走「複製＋雜湊驗證＋刪除來源」，不用未經驗證的直接搬移
- 非圖片檔案與處理失敗的檔案會分別隔離存放，不會混進正常分類結果

## DirectoryFlattener — 目錄扁平化

**解決什麼問題**：多層巢狀資料夾（A/B/C/檔案.txt）想直接攤平成單層，方便集中檢視或後續處理。

**使用情境**：
- 資料夾結構過深，想簡化成單層再做下一步整理
- 巢狀資料夾中有重複檔案，攤平時想順便去重

**核心設計**：
- 兩階段執行：先全部複製到目標層並雜湊驗證，全部通過才進入刪除來源與清空空資料夾的階段
- 第一階段任何一個檔案驗證失敗，會觸發整批 Rollback，刪除本次已複製的檔案，來源保持原封不動
- 內容完全相同的撞名檔案視為真重複，只保留一份；內容不同則自動加流水號區分

## AegisArchiver — 資安去毒歸檔

**解決什麼問題**：來路不明的 PDF、下載回來的檔案，想先做基本資安檢查再歸檔，避免夾帶惡意腳本。

**使用情境**：
- 整理從外部（信箱附件、網路下載、隨身碟）收集來的文件
- 需要把可疑檔案跟安全檔案物理隔離，而不是只是重新命名警示

**核心設計**：
- PDF 檢測 `/JS`、`/JavaScript`、`/OpenAction`、`/Launch` 等高風險標籤，命中即判定風險並隔離原始檔案
- 安全的 PDF 會做基礎去毒處理（移除連結內的自動動作）後才存入安全區
- 風險檔案移至隔離區，安全區留下純文字佔位符說明「這裡原本有一個檔案，因為什麼原因被隔離到哪裡」，方便日後追溯
- 非 PDF 格式（圖片、Office 文件）走一般複製歸檔，本工具的資安檢測目前聚焦於 PDF

## VBAHistorySequencer — VBA 版本血緣分析

**解決什麼問題**：手上有一堆 Excel 巨集檔案的歷史備份，檔名看不出誰是誰的舊版本，想反推出演化順序、找出哪些模組是真正獨創、哪些是複製繼承。

**使用情境**：
- 整理多年累積的 VBA 巨集備份，想搞清楚版本脈絡
- 想找出所有備份中真正互不相同的模組（避免保留大量幾乎相同的重複檔案）

**核心設計**：
- 指定一份「最終定稿」作為錨點，其餘檔案依與錨點的模組相似度排序，推算出世代（Gen001、Gen002…）
- 逐模組計算雜湊，標記「突變/首創」與「繼承自其他版本」，並偵測整批模組完全相同的純複製體
- 巨量檔案／解析逾時情境下不做全數回滾，改為保留已處理進度並在報表誠實記錄失敗項目
- 解析失敗（檔案損毀、格式不支援）的檔案會被隔離存放，不會中斷整批處理

## 共通設計原則

這九支工具雖然功能各自獨立，但貫穿了幾個共同的安全底線（詳見各工具程式碼內的 `FORBIDDEN` 宣告）：

- **搬移前必先複製並驗證**：所有涉及「移動」的操作，實際上都是「複製 → 雜湊或大小驗證 → 確認無誤才刪除來源」，中途失敗一律保留原始檔案
- **不做靜默失敗**：解析失敗、驗證失敗都會明確記錄在報表或日誌中，不會不明就裡地跳過或吞掉錯誤
- **可稽核**：每支工具執行後都會產出 CSV 報表或日誌檔，方便事後核對處理結果
- **原始輸入原則上不被破壞性修改**：多數工具採複製到新目錄的方式運作，來源資料夾除非明確設計為「安全移動」，否則不會被直接改動

工具皆以 MIT 授權釋出，歡迎自行取用修改。因為是 AI 輔助、非本科背景寫成，正式重要資料建議先在測試資料夾試跑過一輪，確認行為符合預期後再用於正式資料。
