# 📂 DirectoryFlattener (資料夾扁平化工具)

本工具旨在將指定資料夾內的「所有子目錄檔案」集中提取至該資料夾的根層級（即「扁平化」），並自動刪除已清空的子目錄。

為了確保資料安全，本工具具備 **SHA-256 雜湊比對** 與 **兩階段搬移 (Two-Phase Strategy)** 機制，確保在遇到硬碟容量不足或權限異常時，能最大程度保障原始檔案不遺失。

---

## 🛠️ 環境建置

本腳本秉持「零相依性」原則，**僅使用 Python 內建標準函式庫**，無須安裝任何第三方套件。

1. **系統需求**：安裝 Python 3.6 或以上版本（因使用 f-string 語法）。
2. **驗證環境**：
   打開終端機 (命令提示字元 / PowerShell / Terminal)，輸入以下指令確認 Python 已正確安裝：
   
 ```bash
 python --version

```

*(註：若在 Mac/Linux 環境下，可能需要輸入 `python3 --version`)*
3. **下載腳本**：將本程式碼儲存為 `flattener.py`（或您自訂的檔名），放置於任意位置即可。

---

## 🚀 使用方式

本工具採用互動式 CLI (命令列介面) 設計，不需記憶複雜的參數即可使用。

### 執行步驟

1. **啟動程式**：
在終端機中執行該腳本：
```bash
python flattener.py

```


2. **輸入目標路徑**：
畫面會提示：`請輸入或拖入要拍扁的資料夾路徑:`
* 您可以直接輸入絕對路徑（例如：`C:\Users\Name\Desktop\MyFolder`）。
* **推薦做法**：直接將目標資料夾用滑鼠「拖曳」到終端機視窗內，系統會自動填入路徑（腳本會自動濾除路徑前後的多餘引號）。按下 `Enter` 繼續。


3. **最後確認 (防呆機制)**：
系統會顯示：`確定要拍扁 '您的路徑' 嗎？此操作不可逆！(y/n):`
* 輸入 `y` 並按 `Enter` 正式開始執行。
* 輸入 `n` 則取消操作。


4. **等待執行與查閱報表**：
程式執行完畢後，終端機會顯示執行結果，並在「目標資料夾」內產生兩份稽核檔案。按下 `Enter` 即可關閉視窗。

---

## 📄 產出檔案說明

每次執行結束後，系統會在您指定的目標資料夾內自動生成兩個檔案，用於稽核與溯源：

1. **執行報表 (`flatten_report_YYYYMMDD_HHMMSS.csv`)**
* 記錄每一筆檔案的「原始路徑」、「新檔名」與「執行狀態」。
* 方便使用 Excel 開啟，快速篩選是否所有檔案皆成功處理。


2. **系統日誌 (`flatten_syslog_YYYYMMDD_HHMMSS.log`)**
* 記錄程式執行的完整生命週期，包含 Phase 1 (複製與驗證) 與 Phase 2 (刪除與清理) 的詳細狀態。
* 若發生錯誤或觸發 Rollback，此檔案會記錄詳細的例外原因（例如：權限不足、檔案被佔用等）。



---

## 🛡️ 核心安全機制說明

為了避免資料毀損，本工具內建以下防禦機制：

* **檔名衝突處理**：若遇到同名檔案，會先比對兩者的 SHA-256 雜湊值。
* **內容相同（真重複）**：直接刪除來源檔，保留一份，報表標示 `[重複略過]`。
* **內容不同（假重複）**：自動在檔名後方加上流水號（如 `file_1.txt`），確保兩份檔案皆安全保留。


* **兩階段搬移防護 (Two-Phase Strategy)**：採取「先複製並驗證雜湊 → 確保全部成功後，再刪除來源」的策略。
* **Phase 1 (複製與驗證)**：此階段若中途發生錯誤（如硬碟空間不足、權限異常），系統將觸發 Rollback，自動清除剛複製的半成品，來源檔案將不受任何影響。
* **Phase 2 (來源清理)**：只有在所有檔案皆 100% 複製且雜湊驗證通過後，才會開始刪除來源檔。若在此清理階段發生系統中斷，最壞情況僅是殘留部分原始檔案或空資料夾，絕不會發生「新檔未建、舊檔已刪」的資料遺失風險。
