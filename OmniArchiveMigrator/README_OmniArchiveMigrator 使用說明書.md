# OmniArchiveMigrator (全能歸檔遷移器)
## 判斷檔名品質，用內容特徵重新命名雜亂檔案
這是一支用於自動化整理、分類並安全遷移雜亂資料夾檔案的 Python 腳本。它能自動提取文件內容特徵重新命名無意義的檔名，並具備嚴格的檔案去重、防碰撞與溯源稽核機制。

**版本**: 1.0.0 (Experimental)

**授權**: MIT License

---

## 核心功能

* **智慧命名**: 針對命名無意義的檔案（如 `新建 Microsoft Word 文档.docx` 或 `temp.txt`），自動讀取內容提取前幾字中文作為特徵標籤。
* **無損安全複製**: 全程使用 `shutil.copy2`，保證檔案位元組層級的一致性，絕不破壞原檔或竄改原始編碼。
* **MD5 精確去重**: 遞迴掃描目錄時，透過 MD5 雜湊辨識內容完全相同的檔案，避免浪費空間。
* **極限防碰撞**: 遇到檔名相同但內容不同的檔案，會自動附加 MD5 特徵碼或遞增後綴，確保 0% 覆寫率。
* **Fail-Fast 稽核報表**: 每一筆檔案的處理狀態皆會記錄於記憶體佇列，即使程序中途中斷（含使用者按 Ctrl+C），已處理的紀錄仍會在結束前完整寫入 CSV 報表，確保可溯源。

## 環境需求與相依套件

請確保您的環境安裝了 Python 3.8 或以上版本，並執行以下指令安裝必備的解析套件：

```bash
pip install python-docx openpyxl pypdf python-pptx

```

> *(註：若未安裝上述套件，腳本仍可執行，但將無法提取 Office 與 PDF 檔案的特徵內容進行智慧命名。)*

---

## 使用說明

1. 將 `Script_OmniArchiveMigrator.py` 存放於任意目錄。
2. 開啟終端機或命令提示字元，執行腳本：
```bash
python Script_OmniArchiveMigrator.py

```


3. 貼上您想要進行整理的「來源資料夾」路徑。
4. 腳本會在來源資料夾的同一層級，自動建立一個帶有時間戳記的 `GOD_ULTIMATE_VAULT_...` 資料夾，內部包含：
* `美名保存區/`: 原命名已具備意義的檔案，原名保存。
* `流浪漢隔離區/`: 原命名無意義的檔案，經提取特徵後重新命名。
* `00_全能歸檔診斷報告.csv`: 完整溯源報表。

## 已知限制 (NOTES)

在以下幾種情境下，特徵提取探針會回傳 **「內容辨識失敗」**，此為預期內行為，不影響檔案本身的完整複製與保存。發生原因將記錄於報表的「備註」欄：

1. **Office / PDF 首頁空白**：探針預設只抓取文件最前方的文字。
2. **純圖片文件**：文件中若沒有可選取的純文本（如全圖片的掃描檔 PDF）。
3. **密碼保護**：受密碼保護的檔案無法透過常規手段解開讀取。
4. **極端編碼破壞**：TXT 檔案的編碼若嚴重損毀，導致所有標準編碼（包含 UTF-8 與 Big5）皆無法強制讀取時。
