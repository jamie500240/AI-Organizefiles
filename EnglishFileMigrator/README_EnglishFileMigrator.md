# 📂 Script_EnglishFileMigrator (英文檔案自動化移轉系統)
## 依內容語系自動把英文檔案分流到獨立資料夾


**版本:** v2.0.0 [Experimental]

**核心功能:** 自動掃描指定目錄，分析檔案內容語系（支援 PDF, DOCX, XLSX, PPTX 等多種格式）。若判定為英文檔案，將安全移轉至專屬資料夾（包含 SHA-256 雜湊值防呆驗證），並產出完整的溯源對照報表。

---

## 🛠 一、 環境建置 (Environment Setup)

本系統依賴 Python 環境與第三方解析套件。若需啟用「圖片文字辨識 (OCR)」功能，則需要額外安裝系統級別的 Tesseract 引擎。

### 1. 基礎環境需求

* **作業系統:** Windows / macOS / Linux
* **Python 版本:** Python 3.8 或以上版本

### 2. 安裝 Python 依賴套件

請開啟終端機 (Terminal / 命列提示字元)，在系統中執行以下指令來安裝解析文件所需的套件：

```bash
pip install python-docx pdfminer.six python-pptx openpyxl

```

### 3. 進階環境建置：啟用圖片 OCR 辨識（選用）

如果您希望系統能夠辨識圖片（JPG, PNG 等）內的文字來判定語系，請完成以下**兩個步驟**。若不需此功能可跳過。

**步驟 A：安裝 OCR 關聯的 Python 套件**

```bash
pip install pytesseract Pillow

```

**步驟 B：安裝 Tesseract-OCR 軟體引擎 (非常重要)**
`pytesseract` 只是 Python 的呼叫橋樑，您必須在電腦上安裝真實的 Tesseract 軟體：

* **Windows:**
1. 前往 [Tesseract 官方 Windows 下載頁面](https://www.google.com/search?q=https://github.com/UB-Mannheim/tesseract/wiki) 下載安裝檔。
2. ⚠️ **安裝時注意：** 在選擇元件 (Choose Components) 的步驟中，展開 `Additional language data (download)`，**務必勾選 `Chinese Traditional` (繁體中文)**。若未安裝中文包，遇到中文圖片時會辨識為亂碼，導致語系判定失效。
3. 安裝完成後，請確保 Tesseract 的安裝路徑（通常為 `C:\Program Files\Tesseract-OCR`）已加入系統的環境變數 (PATH) 中。


* **macOS:**
```bash
brew install tesseract tesseract-lang

```

## 🚀 二、 使用方式 (Usage Instructions)

### 1. 啟動程式

在終端機或命令提示字元中，切換到腳本所在的目錄，輸入以下指令啟動：

```bash
python Script_EnglishFileMigrator.py

```

### 2. 互動式操作流程

啟動後，系統會依序提出三個問題，請根據您的需求輸入：

1. **請輸入目標資料夾路徑:**
* 輸入您要掃描的根目錄路徑（例如：`D:\Projects\Contract_Files` 或直接將資料夾拖曳進視窗中）。


2. **是否啟用圖片內容 OCR 辨識？[y/N]:**
* 輸入 `y`：系統會嘗試讀取圖片內的文字來判定語系（需已完成上述 OCR 環境建置）。
* 輸入 `N` 或直接按 Enter：跳過圖片內容掃描。


3. **是否允許移轉圖片檔案？[y/N]:**
* 輸入 `y`：判定為英文的圖片將會被移動。
* 輸入 `N` 或直接按 Enter（預設）：**安全排除機制**。為避免打散專案原本的相簿或資料夾結構，圖片即使是全英文也將「保留原位」。



**執行畫面範例：**

```text

是否啟用圖片內容 OCR 辨識？(需安裝 tesseract) [y/N]: N
是否允許移轉圖片檔案？(預設不搬移，以保護專案相簿與資料夾結構) [y/N]: N
```

## 📂 三、 產出結果與安全機制說明

執行完畢後，您會在您輸入的「目標資料夾」內看到以下產出：

1. **`英文區` (資料夾):**
* 所有被判定為英文的檔案皆會安全移轉至此。
* **安全防禦機制：** 系統採用 `SHA-256` 雜湊比對，在確認「英文區」的檔案與原始檔案的 Byte 完全一致後，才會刪除原位置的檔案，杜絕因硬碟壞軌或傳輸中斷導致的檔案損壞。


2. **`英文檔案移轉對照表.csv` (報表):**
* **溯源必備：** 紀錄了每一個檔案的「原始位置」、「目前位置」、「判定結果 (英文/非英文)」以及「動作狀態 (已移轉/保留原位/異常)」。
* 若遇系統崩潰或強制中斷，此報表會安全存檔為 `.crashed` 格式，確保已經移轉的軌跡不會遺失。



### 💡 附錄：語系判定規則

系統採用雙軌機制以確保精準度：

* **短文檔（小於 50 字元）:** 中文字數小於 10 字即視為英文（容忍檔名或微量標題帶有中文）。
* **長文檔（大於等於 50 字元）:** 內文的中文字元數必須 **小於總字數的 5%**，以解決長篇英文合約因含有公司中文抬頭而遭誤判為非英文的問題。
