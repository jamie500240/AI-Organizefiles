# 📁 Script_MediaCollapseMigrator (自動整理圖片)
## 大量照片依日期/人物分類，並拍平瑣碎的空目錄
**版本**: 1.0.0 [Experimental]

**用途**: 自動掃描指定資料夾，提取圖片屬性（時間戳記、AI 生成熵值分析），將檔案安全移轉至分發目錄。同時執行「貪婪坍塌（Greedy Collapse）」演算法平攤無意義的深層空目錄，並強制生成完整的資料遷移溯源對照表（Traceability Report）。

## 核心特性

1. **安全移轉 (Safe Move)**：嚴格遵守「複製 ➔ 大小與 MD5 驗證 ➔ 刪除原檔」的原則，拒絕未經驗證的搬移，徹底杜絕斷線或 IO 錯誤導致的資料遺失。
2. **原子性回滾 (Atomicity)**：目錄坍塌過程中若發生單一檔案讀寫失敗，將觸發自動回滾機制，將該批次已移動的檔案安全退回原位，維持檔案狀態一致性。
3. **圖片 DNA 解析與 AI 判定**：透過 EXIF 與檔案屬性萃取真實日期，並結合灰階直方圖熵值（Entropy）演算法，標記潛在的 AI 生成圖片。
4. **容錯與隔離 (Fail-Fast)**：遭遇損毀或無法讀取的檔案時不會靜默吞沒例外，而是明確記錄錯誤，並將問題檔案統一隔離至 `[FAILED_TO_PROCESS]` 目錄。

## 環境建置 (Environment Setup)

### 1. 系統需求

* **作業系統**：Windows / macOS / Linux (需具備圖形介面以支援 Tkinter 資料夾選取視窗)
* **Python 版本**：Python 8 或以上

### 2. 安裝相依套件

本模組依賴部分外部套件進行影像處理與進度顯示。請在終端機執行以下指令進行安裝：

```bash
pip install Pillow numpy tqdm

```

*注意：若您使用 Linux 系統，可能需要額外安裝系統級別的 Tkinter 函式庫（例如 Ubuntu 需執行 `sudo apt-get install python3-tk`）。*

## 使用說明 (Usage)

### 執行程式

1. 在終端機或命令提示字元中執行：
```bash
python Script_MediaCollapseMigrator.py

```


2. 程式會彈出一個資料夾選擇視窗。
3. 選擇您要進行「坍塌重構」的**來源資料夾**。
4. 程式將自動執行 DNA 採集、分發與目錄坍塌。執行完畢後，請按下 `Enter` 結束程式。

### 產出結構

程式不會直接修改原始資料夾名稱，而是會在來源資料夾的**同一層目錄**下，建立一個帶有時間戳記的新資料夾 `[MEDIA_VAULT]_HHMMSS`。其內部結構如下：

```text
[MEDIA_VAULT]_143022/
├── [ASSETS]/                  # 非圖片檔案（影片、文件等）的隔離區
├── [FAILED_TO_PROCESS]/       # 讀取失敗、權限不足或損毀的檔案隔離區
├── 20260715/                  # 依日期重構的資料夾 (少於設定張數的目錄會被坍塌平攤)
├── 人工標籤資料夾/            # 保留原本具備人類命名意義的第一層目錄
└── Migration_Traceability_143022.csv  # 遷移溯源對照表 (強制生成)

```

### 參數微調 (CONFIG)

若需調整判定邏輯，可直接修改原始碼頂部的 `CONFIG` 字典：

* `THRESH_AI` / `ENTROPY_AI`：調整判定是否為 AI 產物的數學熵值閾值。
* `LV1_DATE_MIN`：孤兒日期夾的坍塌標準。預設為 `2`，即資料夾內若少於 2 張圖片，該資料夾將被破壞，檔案會被往上層平攤。
* `MAX_DIR_LEN` / `MAX_FILE_LEN`：動態截斷過長的目錄或檔案名稱，避免作業系統路徑長度限制報錯。

## 限制與注意事項 (Limitations & Known Issues)

1. **磁碟空間需求 (重要)**
由於本系統採用最高安全級別的 `safe_move` (複製 ➔ 驗證 ➔ 刪除)，在搬移任何單一檔案的瞬間，目標硬碟**必須具備大於該單一檔案大小的可用空間**。系統不支援在剩餘空間為 0 的極端滿載硬碟上進行原地搬移。
2. **無周邊環境 (Headless Environment) 限制**
本腳本目前依賴 `tkinter.filedialog` 進行來源目錄選擇。若需在無圖形介面（GUI）的伺服器環境或 CI/CD 流程中透過 Crontab 執行，需自行將 `filedialog` 改為使用 `sys.argv` 接收命令列路徑參數。
3. **AI 判定為啟發式預測**
模組判定檔案是否為 AI 生成（檔名加入 `_A` 標籤），是依賴像素灰階分佈的資訊熵（Entropy）作為啟發式判斷，並非具備 100% 準確率的機器學習模型。高對比度的真實極簡攝影照片亦有可能被誤判。
4. **檔名截斷與碰撞**
為避免超長檔名報錯，系統會過濾特殊字元並截斷檔名（預設保留 60 字元）。若極端情況下有兩份檔案時間戳記相同、檔名前 60 字元完全相同，可能會在目標目錄產生命名碰撞（系統會透過加入 timestamp 後綴來自動迴避，但檔名會產生變動）。
